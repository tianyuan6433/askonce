Pivot Plus Product Training
Hi everyone! Today I’m gonna break this training into 4 parts. First, why we built Pivot Plus and its key upgrades. Second, how we’ll sell it. Third, the main customer questions we might get. Let’s jump right in!
Pivot Plus came from fixing problems we saw with the old Pivot. Over the years, we’ve worked with 3,000+ organizations and 15,000+ devices. But we ran into issues: device enrollment fails a lot, slowing things down. Remote screen control keeps disconnecting. Firmware updates are late—we had to manually install firmware via USB for lots of devices, which is super inefficient. Also, new hardware comes out but software can’t keep up.
The big reasons? First, the old Pivot’s base wasn’t stable enough, so it didn’t work well with new gear—like Microsoft Teams Room devices or Google’s EDLA service for education. So we redesigned it to be more flexible and compatible. That’s Pivot Plus!
We’ve spruced up the whole experience: the design is more modern and clean. Menus are simpler but still look sharp. Every feature—from enrollment and management to maintenance and future CMS paid services—focuses on saving time. We’re also getting more safety certifications under MAXHUB. SOC 2 is in progress, and we should get the certificate in February.
Pivot Plus is a web-based cloud management platform that combines CMS and DMS. Our CMS is built like popular commercial content distribution tools, covering everything from enrollment and control to maintenance and content display—all to make work faster. We’re still using it for meetings and education, but now we’re trying retail too, with better CMS content display as the key.
Let me show you the main features. The interface has 5 main sections: Dashboard, Devices, Publish, System, and logs. Dashboard shows all device tasks, status, and basic data. Devices has all device management and maintenance tools. Publish handles all related tasks. System has settings, and all logs are in one place called “Logs.”
First, device enrollment and connection—we upgraded the zero-touch enrollment:
Remote pre-import: For MTR-Android devices, you used to need to finish setup before using them. Now, you can import the device’s SN and Mac info remotely, plus preset OOBE settings (like language, time zone, OTA). Once the device turns on, it auto-completes setup and connects to the platform.
Same-network detection: If your organization already has some connected devices, the system uses their IP, router’s Mac address, or Wi-Fi SSID to find other devices on the same network. Just click “Enroll” to connect them quickly.
I just connected an IFP device and an MTR Android device—you can see them in the device list now.
These connected devices support one-click batch remote control. Both the MTR’s bar screen and console screen can be managed together. Watch—my actions show up on the console right away. For IFP devices connected to the bar, I can also see and adjust controls in real time.
Now the CMS you’ve been waiting for! It has 4 parts: resource management, canvas creation, playlist setup, and schedule planning. After uploading materials, you can see storage usage and file types at the top. The canvas lets you make quick edits—add videos, pictures, text, or QR codes. For example, I can change the canvas background to green, add a MAXHUB promo video, and a QR code linking to baidu.com. Once done, publish it to a device instantly.
We use offline playback: looped tasks download materials first, then delete them after playing. This stops devices with small storage (like CMA) from running out of space. Playlists let you drag and drop single files, canvas creations, or old playlists—easily adjust the order and playtime. Schedules are also drag-and-drop: move a weekly task from Tuesday/Thursday to Wednesday/Friday, or change a date from the 23rd to the 24th. You can check if all tasks worked in the history and logs.
We also upgraded maintenance:
Anomaly detection: Monitor 10+ issues (like low disk space). Set alerts and choose which devices to watch. Alerts go to managers via in-app messages or email—you can fix problems remotely.
Firmware updates: Now update peripherals (like MTR consoles) too. For urgent temporary firmware not on the OTA platform, upload it manually and push it to all devices.
Dashboard: Blue = online and normal, light blue = offline, orange = needs attention. Click to see device screens and details, plus view past, present, and future tasks. Export data easily.
Those are the 4 key upgrades: easier/faster enrollment , multi-device remote control, clear dashboard, and better remote maintenance + content display.

Pivot Plus 产品培训讲稿
各位同事，今天的培训主要分为四个部分：一是 Pivot Plus 的产品背景与升级价值，二是核心功能升级展示，三是销售策略与版本划分，四是里程碑计划与迁移相关说明，最后会解答大家的疑问。
首先，我们来聊聊为什么要推出 Pivot Plus 这款产品。此前 Pivot 在服务 3000 多个组织、5000 多台设备的过程中，暴露了不少运维问题：设备注册频繁失败，影响整体效率；远程桌面控制容易断连；固件更新不及时，临时固件无法批量下发，设备量大时只能靠运维人员手动 USB 刷机；还有硬件上市后软件适配跟不上的情况。这些问题的核心原因，一是原有产品底座稳固性不足，兼容度不高；二是互联网环境瞬息万变，我们推出了 MPR 类会议产品、谷歌教育 FP 环境的 edla 服务等新型硬件，而早期产品并未考虑到这些特殊适配需求。
所以，为了兼顾更多诉求，打造更灵活、兼容性更强的产品力，我们在原有 Pivot 的基础上，借鉴运维管理经验，升级推出了 Pivot Plus。这款产品在整体体验上做了全面优化：视觉风格更现代化、干净整洁，菜单以轻量化方式呈现，增强视觉冲击感；从注册、管控、运维到后续 CMS 商业化服务，全流程以提效为核心进行管理升级；同时我们以 MAXHUB 名义申请更多安全认证与合规证书，目前 SOC 2 认证正在进行中，预计 2 月份就能拿到证书。
Pivot Plus 的核心定位是基于 Web 平台的云管理平台，整合了 CMS 与 DMS 两大能力。其中 CMS 对标线上主流商用内容分发平台构建，提供从注册、控制、运维到信发的全流程提效服务。适用场景上，除了原有会议、教育场景，我们还新增了零售方向探索，核心发力 CMS 信发能力。
接下来给大家演示产品的核心功能升级，大家可以看到当前的产品界面，整体功能分为四大模块：总览、设备、信发服务、系统。总览模块能查看所有设备的任务、状态及基础数据；设备模块整合了所有设备管理与运维相关能力；信发服务模块集中管理所有信发相关操作；系统模块包含各类设置项，平台所有日志统一归集在 logs 中。
先看设备注册与关联功能，我们对原有零接触注册做了两大提升：一是远程预导入，大家熟悉的 MTR 设备，出厂后原本需要完成开机引导才能使用，现在通过 Pivot Plus，可远程提前导入设备 SN+Mac 信息，预设 OOBE 配置文件，包含语言、时区、OTA 等信息，设备开机后会自动执行 OOBE 任务，完成后直接关联至平台设备列表。二是同网络探测关联，若组织已绑定部分设备，系统会获取这些设备的 IP、路由器 Mac 地址（或 Wi-Fi 的 SSID），以此探测同网络下未关联的设备，点击 enrollment 就能快速完成关联。我现场就关联了一台 FP 设备和一台 MTR 安卓设备，大家可以看到它们已成功出现在设备列表中。
这些关联后的设备支持批量远程一键控制，无论是 MTR 设备的 bar 画面还是 console 画面，都能通过联集的方式进行管控。大家可以看到，我的操作能实时同步到 console 上，对于 bar 连接的 FP 设备，也能实时查看控制动作、调试状态并进行调整，响应非常及时。
再重点说说大家期待的 CMS 信发功能，整体框架分为资源素材管理、画布创作、播放列表排布、日程排程四部分。素材上传后，顶部会显示资源占用情况及不同类型资源信息；画布创作支持快速创建，可编辑视频、图片、文字、二维码等内容，比如我把画布底色调成绿色，加入 MAXHUB 宣传片，再添加一个指向baidu.com的常驻二维码，创作完成后能实时发布到指定设备。播放方式采用离线播放，循环任务会提前下载素材，播放完成后自动清理本地资源，避免 CMA 这类存储空间不足的设备出现存储爆满问题。播放列表支持单素材、画布创作内容、原有播放列表的拖拽添加，顺序和播放时间可灵活调整；日程排程也支持拖拽修改，比如把周二、周四的周循环日程调整到周三、周五，把 23 号的定时指令移动到 24 号，操作非常便捷。所有信发任务的执行情况，都能在历史记录和指令日志中查看。
运维部分的升级也很实用，首先是设备异常事件管理，支持对磁盘空间不足等十多种特殊事件进行定时探测，可设置告警阈值、关注设备及告警级别，通过站内信和邮件推送给管理者，待解决的异常情况能在平台查看，还能直接远程处理。其次是固件升级，以前只能对主设备升级，现在所有外设（比如 MTR 的 console）都能快速选择类型、查看更新状态；对于未及时同步到 OTA 平台的紧急临时固件，可手动上传并批量发布给所有设备，解决临时性问题。最后是工作台优化，蓝色标识在线正常设备，浅蓝色是离线设备，橙色是有异常需处理的设备，点击可查看设备屏幕画面和基础详情并进行管理；任务以视图形式呈现当日、历史及未来需执行的任务，所有指令集中展示，设备使用、软件使用等基础数据可快速查看和导出。
以上就是核心更新的四大能力：更简洁高效的注册方式、多设备远程控制、清晰直观的工作台、强化的远程运维与信发功能。
