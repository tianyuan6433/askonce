MANAGE  4.1 DEVICE CONTROL 4.1.1 POWER ON There are two ways to wake up a device: 4.1.1.1 Time mode: execute now, timed and recurring Commands will be saved on the server, and the device will receive the command at the set time. It relies on the build of the Wake On LAN function. The device can wake up when connected to a Bytello WOL service. See Chapter 11 for more information on Bytello WOL Service. 4.1.1.2 Time mode: recurring with oﬄine mode Commands will be saved locally after the device receives them for the ﬁrst time. Support for oﬄine line mode.  See chapter 3.3 for more information on Oﬄine mode. 4.1.1.3 Power on via the online device Starting with version 23, there's a new way to wake up your device! Support the online devices as WOL Services. After an online device receives the power-on command, it can send command to the devices in the shutdown state via LAN to power on these devices. Previously, it was necessary to download and deploy a WOL server on a Windows device to allow devices on the same network to boot up. Now, every online device can send commands to other devices to turn them on. Ensure the below conditions is met: The device must be under the same wired LAN. At least one of the devices receiving the Power on command must be online to wake the other devices. The client version must be 23 or later. 4.1.2 POWER OFF The device will display a pop-up window for conﬁrmation when the power-oﬀ command is sent. The user in front of the device can reject the Power oﬀ command this time.  If there is no user action for the 60s, the Power oﬀ will be performed. 4.1.3 REBOOT When the restart command is sent, the device will display a pop-up window for conﬁrmation. The user in front of the device can reject the Reboot command this time.  If there is no user action for the 60s, the Reboot will be performed. 4.1.4 BELLS Upload local upload of audio ﬁles and then set a play time. 4.1.5 SCREEN LOCK Screen lock can lock a speciﬁed device for a period of time.
